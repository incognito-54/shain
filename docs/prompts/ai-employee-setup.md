# AI社員システム構築プロンプト(Hermesに1回送る初期構築用)

【目的】
このリポジトリ(Hermes)の上に「AI社員」の仕組みを作りたい。
AI社員とは: 職務記述書を持ち、特定の業務(例: 週次レポート作成、契約書の転記、調査代行)を
定期実行またはワンショット実行(npm run task)で継続的に担当するロール型エージェントのこと。
新しい人格やモデルを作るのではなく、「職務の定義ファイル一式 + 既存ワーカーがそれを読んで演じる」方式で実現する。

【前提知識(このリポジトリの構造)】
- src/agents.ts にワーカー定義(researcher/coder/reviewer/writer)がある
- npm run task -- "依頼文" でワンショット実行できる(cron連携可)
- あなた(オーケストレーター)は自分でファイル編集できないので、作成作業はcoder/writerに委譲すること

【やること】
1. docs/AI_EMPLOYEE_MANUAL.md を作成する。内容:
   - AI社員の定義と構成要素
   - 構築手順(新しいAI社員を1体作るときにあなたが従う工程: ヒアリング→JD作成→SOP作成→試運転→検証→運用開始)
   - 運用方法(定期実行のさせ方、報告の読み方、memory.mdの更新ルール)
   - 安全規則(外部送信・削除・課金操作は必ず人間の承認を得る、秘密情報はファイルに書かない)
2. employees/_template/ に雛形一式を作成する:
   - JOB.md(職務記述書: 役割 / 責任範囲 / 完了条件 / 報告形式 / してはいけないこと)
   - SOP.md(業務手順書: タスクごとの具体的手順)
   - RUN.md(実行指示書: npm run task から毎回読み込まれるエントリポイント。JOB/SOP/memoryを読んで業務を実行し、out/ に成果物と報告を書く指示)
   - memory.md(業務で学んだことの蓄積場所)
3. 動作確認として、サンプルAI社員「repo-reporter」(このリポジトリの変更を要約して週次報告を書く係)を
   employees/repo-reporter/ に作成し、実際に業務を1回実行させて employees/repo-reporter/out/ に報告が出ることを確認する
4. reviewerに全体を検証させる(マニュアルの手順どおりに新しいAI社員を作れる状態になっているか、の観点で)

【完了条件】
- マニュアル・テンプレ・サンプルAI社員が存在し、サンプルの業務実行成果物が out/ にある
- 「employees/_template をコピーして穴埋めすれば2体目が作れる」状態になっている
- 最後に、2体目を作るときに私が送るべき依頼文の例を報告に含める

【制約】
- src/ 配下のコードは変更しない(ファイルベースの仕組みだけで実現する)
- 外部サービスへの送信や破壊的操作はしない
- すべて日本語で書く
