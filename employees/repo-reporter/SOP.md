## T1: 今週の変更点の収集

### 手順

1. `git log --since="7 days ago" --oneline --stat` で直近1週間のコミット一覧と変更ファイルを取得する
2. `git shortlog -sn --since="7 days ago"` でコミット件数の集計を確認する
3. `git status --short` で現在の未コミット変更(作業中のファイル)を確認する
4. コミットが少ない/ない場合は、その事実を明記した上で、代わりに現在の未コミット変更や
   リポジトリの構成(ディレクトリ構成、最近更新されたファイルの一覧など)を要約対象にする
5. 変更点を「主な変更」「新規追加」「注意点/リスク」の3種に分類してメモする

### 使うツール/ワーカー

- Bash(git log, git status, git shortlog, find 等)
- 差分の技術的な調査が必要な場合は researcher ワーカーに委譲してもよい

### 注意点

- git リポジトリでない/コミットが取得できない場合はエラー終了せず、「コミット履歴なし」として
  その旨を報告書に明記し、代わりにファイルの更新日時(`find -mtime` 等)やディレクトリ構成から
  最近の変更を推測して報告する
- 秘密情報(.env の中身など)は絶対に読み取って報告に転記しない

## T2: 週次報告の作成

### 手順

1. T1で集めた情報をもとに、JOB.md の「報告形式」に沿って本文を組み立てる
   (対象期間・結論サマリ → 主な変更 → 新規追加 → 注意点/リスク → 来週への申し送り → 参照)
2. 専門用語は残してよいが、簡潔で分かりやすい日本語で書く
3. 報告の冒頭に「本報告は repo-reporter によって自動生成された」旨を1行明記する
4. `employees/repo-reporter/out/report-YYYY-MM-DD.md` として保存する
5. 業務で得た知見・注意点があれば `employees/repo-reporter/memory.md` に日付見出しを付けて追記する

### 使うツール/ワーカー

- Write(ファイル出力)
- Edit(memory.md への追記)

### 注意点

- 実行日の日付を必ずファイル名・本文の両方に反映する
- git push、外部送信、ファイル削除、`src/` 配下の変更は行わない
