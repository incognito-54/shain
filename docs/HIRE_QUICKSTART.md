# AI社員の雇い方 — かんたんガイド

このガイドは、プログラミングの知識がなくても **AI社員を自分で雇って働かせる** ための手順です。
覚えるコマンドは実質2つだけです。

---

## AI社員とは

決まった作業(毎週の報告、書類の転記、情報収集など)を、頼めば何度でも
同じ品質でやってくれる「係」です。1体ずつ、担当業務を決めて雇えます。

---

## ステップ1 — 準備(最初の1回だけ)

ターミナルでこのフォルダを開き、次を1回だけ実行します。

```bash
npm install
```

> うまく動かない・パスワードを聞かれるなどのときは、このシステムは自動でログイン方式に
> 切り替わるようになっています。それでも進まない場合は、詳しい人か開発者に
> 「起動できない」とだけ伝えてください。

## ステップ2 — 雇う

```bash
npm run hire
```

これだけで採用の面談が始まります。画面の質問に、ふだんの言葉で答えるだけです。
聞かれるのは次のようなことです(専門用語は出てきません):

1. どんな作業を任せたいか
2. どのくらいの頻度でやらせたいか
3. 「これができていればOK」と言える状態は何か
4. 絶対にやらせたくないことはあるか
5. 成果物をどこで受け取りたいか

答え終わると、AI社員が自動で作られ、**試しに一度働いた成果物**まで用意されます。
最後に「◯◯係を雇いました。動かし方はこれです」と、コピーして使える1行が表示されます。

## ステップ3 — 働かせる

雇い終わりに表示された1行を、そのままコピーして実行すればいつでも働きます。例:

```bash
npm run task -- "employees/repo-reporter/RUN.md を読んで業務を実行して"
```

成果物は、その係の `employees/<名前>/out/` フォルダに置かれます。

> **毎週など自動でやらせたい**場合は、雇うときに「毎週月曜の朝に自動で」と伝えれば、
> 自動実行の設定までやってもらえます。

---

## よくある質問

- **何体でも雇えますか?** はい。`npm run hire` を実行するたびに1体ずつ増やせます。
- **後から仕事内容を変えられますか?** できます。`npm run hire` の代わりに、ふだんの
  対話画面(`npm run start:login`)で「◯◯係の仕事に△△も足して」と頼んでください。
- **危ないことをしませんか?** メール送信・データ削除・お金を使う操作は、あなたの許可なしには
  行わない設計です。雇うときに追加の禁止事項も指定できます。

困ったことや「こんな係が欲しい」という相談は、対話画面(`npm run start:login`)で
そのまま日本語で話しかけてください。
