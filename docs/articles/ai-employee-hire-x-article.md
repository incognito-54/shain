## 本文

# コードを書かずに「AI社員」を雇える仕組みを作った — 経営者が自分で、面談に答えるだけで

## きっかけ:「もう1人いたら」を、人を増やさずに叶えたい

事業をやっていると、「この作業、誰かに任せられたら」と思う瞬間が毎日あります。
毎週のレポート、経理の記帳、案件やニュースの巡回、書類の転記。どれも判断は難しくないのに、
手数が多くて時間を吸われる。かといって、そのために人を1人雇うほどではない。

AIに任せればいい——のはわかっているけれど、いざ自分の業務にAIを合わせ込もうとすると、
プロンプトを書いて、ツールをつないで、と急に“エンジニアの仕事”になってしまう。
経営者がやりたいのはそこじゃない。

なので、**経営者が自分で、コードを書かずに「AI社員」を雇える**仕組みを作りました。

## AI社員とは — 「決まった仕事を、頼めば何度でも同じ品質でやる係」

AI社員は、担当業務を1つ持った“係”です。人格やモデルを新しく作るわけではありません。
「何を、どうやって、何をもって完了とするか」を決めた**業務マニュアルを持ったAI**、と考えてください。

一度雇えば、コマンド1つで何度でも同じ品質で働きます。放っておけば毎週月曜の朝に
勝手に走らせることもできます。

## 雇い方:ターミナルで `npm run hire`、あとは面談に答えるだけ

いちばん見てほしいのがここです。AI社員を増やすのに、設定ファイルもプロンプトも要りません。

`npm run hire` と打つと、AIが**採用面談**を始めます。聞かれるのは、専門用語のない、ふつうの質問だけ:

1. どんな作業を任せたいですか?
2. どのくらいの頻度でやらせますか?
3. 「これができていればOK」と言える状態は?
4. 絶対にやらせたくないことはありますか?
5. 成果物はどこで受け取りたいですか?

普段の言葉で答えていくと、AIが裏側で職務マニュアル一式を組み立て、**試しに一度働いた成果物**まで
用意して、「◯◯係を雇いました。次はこう動かします」と、コピーして使える1行を返してくれます。

面接官(AI)が新人(AI)を採用して、そのまま研修まで済ませてくれる——そんな感覚です。

## 一番こだわったのは「させないこと」の設計

AIに仕事を任せると聞くと不安になりますよね。私もそうでした。なので機能より先に**禁止事項**を設計しました。

- **お金を動かす操作(支払い・振込・購入)は、本人の許可なしに絶対にやらせない。**
- **メール送信・データ削除など、取り消せない操作もやらせない。** AIの仕事は「準備」まで。実行ボタンは人間が押す。
- **パスワードや口座番号などの秘密は、成果物に書かせない。**
- 雇うときに、その社員固有のNG(「この取引先には触れない」等)も追加で指定できます。

つまり「優秀だけど権限のない新人」として雇う。これが、安心して任せられる唯一の形だと思っています。

## 仕組み(興味のある人向けに、少しだけ)

中身は Claude Agent SDK で組んだ、二層構造のマルチエージェントです。

- **対話層**が1体。あなたの窓口になり、依頼を理解して段取りするだけ。自分では重い作業をしない。
- **作業層**が複数体。調査・実装・検証・文書作成・ブラウザ操作を、必要なものだけ並列で実働する。

そして「AI社員」は、この作業層が読む**数枚のマニュアル(Markdown)**として表現されています。
仕組み(プログラム)と、役割(マニュアル)を分けたので、社員を何人増やしてもプログラムは一切
書き換えません。だから、非エンジニアでも面談に答えるだけで1人増やせる、という形が成立します。

育てるのも簡単です。「経費のメモには相手と目的を必ず書いて」と思ったら、マニュアルに1行足すだけ。
プロンプトエンジニアリングというより、新人に渡す業務手順書を改訂する感覚に近い。

## すでに働いている社員たち

この基盤の上で、実際にいくつかの係が動いています。

- **経理秘書** — 領収書の整理、複式簿記での記帳、月次締め、確定申告の準備まで。ただしお金には一切触らせない。
- **案件スカウト** — クラウドソーシングの新着案件を巡回し、条件に合うものを拾ってスコア付けして報告。
- **週次レポート係** — 決まった情報源の変化を要約して、毎週の状況報告を書く。

同じ型なので、面談を1回やるたびに、こういう係が1人ずつ増えていきます。

## なぜ「経営者が自分で作れる」ことにこだわったか

AI活用の一番の壁は、技術ではなく「自分の業務に合わせ込む人がいない」ことだと思っています。
出来合いのAIツールは便利でも、あなたの会社のやり方は知らない。かといって、そのたびに
エンジニアに頼んでいたら、増やすたびにコストと待ち時間がかかる。

だから「**業務を一番わかっている本人が、面談に答えるだけで、その業務専門のAIを増やせる**」形にしました。
AIを“導入”するのではなく、必要なときに“採用”する。人を雇う感覚のまま、AIの手数を足していけます。

## 公開しました

この基盤は OSS として公開しています。`npm install` して `npm run hire` と打てば、
今日から自分専用のAI社員を雇えます。経営者・非エンジニア向けの「かんたんな雇い方ガイド」も同梱しました。

「人は増やせないけど、手は増やしたい」——そういう人にこそ触ってほしいです。
